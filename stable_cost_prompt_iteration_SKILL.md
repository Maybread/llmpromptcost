# Stable-Cost Prompt Iteration Skill

## Purpose

Use this skill to run a cost-function-guided iterative prompt optimization workflow for AI-generated scientific explanations. This version uses a full-score target point and gives equal weight to all five cost components, so it balances stability, full-score proximity, density peak quality, normative-region entry, and weak-dimension repair.

This skill is for experiments where responses have already been scored on three dimensions:

- D1: core concept / energy transformation
- D2: conditional model / ideal vs. real situation distinction
- D3: terminology use / scientific expression

The skill does not train, fine-tune, or update model parameters. It evaluates prompt conditions and generates next-round prompts based on actual cost-function results.

---

## Core Design

The target point is:

```text
v* = (1.00, 1.00, 1.00)
```

This represents the ideal scientific explanation in which all three dimensions reach the highest level.

The normative region is:

```text
Omega = {x | D1 >= 0.80, D2 >= 0.80, D3 >= 0.80}
```

This represents the minimum high-quality region. A response enters the normative region only when all three dimensions are at least 0.80.

The stable-cost function is:

```text
J_stable(P) = 0.20*C_disp
            + 0.20*C_norm
            + 0.20*C_peak
            + 0.20*C_region
            + 0.20*C_weak
```

The first cost term, `C_disp`, is computed as a root-mean-square normalized distance rather than a simple arithmetic mean.

A lower `J_stable(P)` indicates a better prompt.

This weighting means that the experiment treats the five cost components as equally important.

---

## When to Use This Skill

Use this skill when the user wants to:

- recalculate prompt performance with target point `(1,1,1)`;
- evaluate prompt performance with equal weights across the five stable-cost components;
- identify prompts that generate high-quality responses stably;
- redesign second-round prompts from first-round cost results;
- compare task-component prompts, demonstration prompts, and hybrid prompts;
- produce a reproducible prompt-iteration workflow.

Do not use this skill when:

- no D1/D2/D3 scores are available;
- the task is model fine-tuning rather than prompt-level evaluation;
- the user wants to predefine second-round prompts before first-round cost results are computed.

## Cost Components

For a prompt `P`, suppose it generates `N` responses. Each response is represented as:

```text
x_i = (D1_i, D2_i, D3_i)
```

### 1. C_disp: Conceptual Dispersion Cost

Meaning: whether responses generated by the same prompt are too scattered in the three-dimensional conceptual space.

Formula:

```text
mu = (mean_D1, mean_D2, mean_D3)
d_i = ||x_i - mu||_2 / sqrt(3)
C_disp = sqrt( mean( d_i^2 ) )
```

Interpretation: lower values indicate a more stable response distribution.

Important caution: do not assume lower dispersion is always better. Extremely low dispersion may indicate over-template responses. Mention this possible side effect in the report.

### 2. C_norm: Normative Distance Cost

Meaning: average distance between all responses and the ideal full-score target point.

Formula:

```text
v* = (1.00, 1.00, 1.00)
C_norm = mean( ||x_i - v*||_2 / sqrt(3) )
```

Interpretation: lower values indicate that responses are closer overall to the ideal scientific explanation.

### 3. C_peak: Density Peak Distance Cost

Meaning: distance between the most frequent response type and the full-score target point.

Formula:

```text
p_peak = most frequent score vector under prompt P
C_peak = ||p_peak - v*||_2 / sqrt(3)
```

Interpretation: lower values indicate that the most common response pattern is close to the ideal response. If `p_peak = (1,1,1)`, then `C_peak = 0`.

### 4. C_region: Normative-Region Failure Cost

Meaning: proportion of responses that fail to enter the high-quality normative region.

Formula:

```text
P_omega = number_of_responses_in_Omega / N
C_region = 1 - P_omega
```

Interpretation: lower values indicate that a larger proportion of responses meet the minimum high-quality criteria.

### 5. C_weak: Weak-Dimension Cost

Meaning: whether one or more dimensions have a systematic weakness.

Formula:

```text
theta = (0.80, 0.80, 0.80)
C_weak = max(0, 0.80 - mean_D1)
       + max(0, 0.80 - mean_D2)
       + max(0, 0.80 - mean_D3)
```

Interpretation: lower values indicate no obvious dimension-level weakness.

## Main Workflow

### Step 1: Load Current-Round Results

Read the current prompt-result file, standardize column names, normalize D1/D2/D3, and remove incomplete rows.

### Step 2: Compute Stable Cost for Each Prompt

For every prompt condition, compute:

```text
N
mean_D1
mean_D2
mean_D3
p_peak_D1
p_peak_D2
p_peak_D3
P_omega
C_disp
C_norm
C_peak
C_region
C_weak
J_stable
```

### Step 3: Rank Prompts

Sort prompts by `J_stable` in ascending order.

Do not select prompts by average score alone. A prompt can have a good mean score but still be suboptimal if its responses are dispersed or its peak is far from `(1,1,1)`.

### Step 4: Diagnose Remaining Problems

For each top prompt, identify the highest and second-highest cost components.

Use these diagnostic rules:

| Highest Cost Component | Diagnosis | Revision Direction |
|---|---|---|
| C_disp | Responses are too dispersed | Add stable explanation structure and coherence constraints |
| C_norm | Responses are not close enough to full-score target | Strengthen complete scientific explanation requirements |
| C_peak | Most frequent response type is not ideal | Clarify full-score response components |
| C_region | Too few responses enter the high-quality region | Add minimum answer requirements |
| C_weak | One dimension is underdeveloped | Repair the weakest D1/D2/D3 dimension |

### Step 5: Select Prompts for Next-Round Revision

Select the top 1–2 prompts with the lowest `J_stable`.

In the current research design, first-round results often suggest two useful bases:

- Task-component prompt: tends to support full-score target and normative-region entry.
- Demonstration prompt: tends to support stable response distribution.

If the recalculated stable cost confirms this pattern, use both as next-round bases.

### Step 6: Generate Next-Round Prompts

Do not predefine next-round prompts before seeing the current cost values.

Generate each next-round prompt from an actual selected prompt. For each selected prompt, generate 3–5 variants:

1. A variant targeting `C_disp` if dispersion is high.
2. A variant targeting `C_norm` or `C_peak` if the distribution is stable but not close enough to `(1,1,1)`.
3. A variant targeting `C_region` if too few responses enter the normative region.
4. A variant targeting `C_weak` if a dimension is below 0.80.
5. A hybrid variant combining the strongest features of the top prompts.

Each generated prompt must include:

```text
source_prompt_id
source_prompt_type
target_cost_component
diagnosed_problem
revision_strategy
expected_effect
possible_side_effect
next_round_prompt
```

## Prompt Revision Strategy Library

These are strategy rules, not fixed prompts. Use them only after cost values are computed.

### If C_disp Is High

Goal: reduce response dispersion and increase stability.

Revision strategy:

```text
Add a stable explanation order: energy transformation -> conservation condition -> air resistance -> terminology summary.
```

Possible wording:

```text
请使用稳定的解释结构：先说明能量转化，再说明机械能守恒的条件，然后解释存在空气阻力时会发生什么变化，最后使用准确的科学术语进行总结。请保持解释连贯，并避免遗漏上述部分。
```

### If C_norm Is High

Goal: move responses closer to `(1,1,1)`.

Revision strategy:

```text
Strengthen full scientific completeness: energy transformation, conservation condition, air resistance, and terminology.
```

Possible wording:

```text
请完整说明重力势能向动能的转化、机械能守恒的条件，以及空气阻力存在时的影响，并使用准确的科学术语。
```

### If C_peak Is High

Goal: move the most frequent response type toward `(1,1,1)`.

Revision strategy:

```text
Make the full-score response pattern explicit.
```

Possible wording:

```text
一个完整的科学解释应包括能量转化、守恒条件、空气阻力影响和准确术语。请将这些内容组织成一个连贯解释。
```

### If C_region Is High

Goal: increase the proportion of high-quality responses.

Revision strategy:

```text
Set minimum required explanation components.
```

Possible wording:

```text
回答至少应包含：重力势能和动能如何变化、机械能在什么条件下守恒、存在空气阻力时能量如何转化，以及准确科学术语。
```

### If C_weak Is High

First identify the weakest dimension.

If D1 is weak:

```text
请重点说明初始状态、重力势能如何减少、动能如何增加，以及二者如何相互转化。
```

If D2 is weak:

```text
请明确区分无空气阻力的理想情况和存在空气阻力的真实情况，并说明机械能守恒的条件。
```

If D3 is weak:

```text
请使用准确的科学术语，如重力势能、动能、机械能守恒、空气阻力、热能和声能。
```

## Recommended Second-Round Design Under the New Formula

After recalculating first-round stable cost with `v*=(1,1,1)`, second-round experiments should usually include:

1. The best task-component prompt replication.
2. The best demonstration prompt replication.
3. Component + coherence prompt: strengthen structure to reduce `C_disp`.
4. Demo + full-score target prompt: keep demonstration stability while moving toward `(1,1,1)`.
5. Component + example hybrid prompt: combine full-score components with example-guided stability.
6. Stability-optimized prompt: explicitly test whether strong structure reduces dispersion.

Only use these if first-round cost results support them. Otherwise, revise according to the actual highest cost components.

## Python Pseudocode

```python
def compute_stable_prompt_cost(df_prompt):
    X = df_prompt[["D1", "D2", "D3"]].to_numpy(dtype=float)
    N = len(X)

    v_star = np.array([1.00, 1.00, 1.00])
    theta = np.array([0.80, 0.80, 0.80])

    mu = X.mean(axis=0)

    disp_dist = np.linalg.norm(X - mu, axis=1) / np.sqrt(3)
    C_disp = np.sqrt(np.mean(disp_dist ** 2))

    C_norm = np.mean(np.linalg.norm(X - v_star, axis=1) / np.sqrt(3))

    peak_counts = (
        df_prompt.groupby(["D1", "D2", "D3"])
        .size()
        .reset_index(name="count")
        .sort_values("count", ascending=False)
    )
    p_peak = peak_counts.iloc[0][["D1", "D2", "D3"]].to_numpy(dtype=float)
    C_peak = np.linalg.norm(p_peak - v_star) / np.sqrt(3)

    in_omega = (
        (df_prompt["D1"] >= 0.80)
        & (df_prompt["D2"] >= 0.80)
        & (df_prompt["D3"] >= 0.80)
    )
    P_omega = in_omega.mean()
    C_region = 1 - P_omega

    C_weak = np.sum(np.maximum(0, theta - mu))

    J_stable = (
        0.20 * C_disp
        + 0.20 * C_norm
        + 0.20 * C_peak
        + 0.20 * C_region
        + 0.20 * C_weak
    )

    return {
        "N": N,
        "mean_D1": mu[0],
        "mean_D2": mu[1],
        "mean_D3": mu[2],
        "p_peak_D1": p_peak[0],
        "p_peak_D2": p_peak[1],
        "p_peak_D3": p_peak[2],
        "P_omega": P_omega,
        "C_disp": C_disp,
        "C_norm": C_norm,
        "C_peak": C_peak,
        "C_region": C_region,
        "C_weak": C_weak,
        "J_stable": J_stable,
    }
```

## Important Cautions

1. All five components have equal weight, so no single component should override the others.
2. A prompt with low dispersion but low D1/D2/D3 quality should not be considered optimal.
3. If a prompt has extremely low `C_disp`, inspect `response_text` for over-template or homogenized answers.
4. Do not predefine next-round prompts before current-round stable cost is computed.
5. Do not overwrite original files.
6. Recalculate first-round results after changing the target point to `(1,1,1)`.
7. Use the phrase `stable-cost prompt iteration` for this revised workflow.
8. Use the phrase `稳定性优先的成本函数引导提示词迭代优化` in Chinese writing.
